Hypertext Markup Language (HTML)

  • The < head > element contains information about the webpage like the tittle.
  • The < body > represents the visible content shown to the user.
  • HTML builds the structure of a webpage.
  • It has basic elemments like; text, image, header, and list elements.
  • Common elements to describe a page: meta-related, content division, and semantic elements.
  • HTML elements are usually composed by an opening and closing tag with angle brackets < > .
  • Metadata or < head > element contains information about accesibility, search engines and performance, it is not for user.
  • The < body > element contains the content that is shown to the user.
  • < ! Doctype html > declaration tells the browser to expect HTML on the page.
  • Inside < HTML > is an attribute of lang= "en", which tells the browser the language for the webpage.

The < head > element contains information about the page. The < tittle > element defines the title for a webpage. The < style > element defines the CSS style.
The < base > element defines the base URL for a webpage. The < link > element connects an external resource to the HTML document. The < meta > element defines description, keywords, author, and viewport.
The < ssript > element embedss client-side scripts in the HTML document.
  • The < body > element contains child elements that are the elements nested in the body element also known as the parent element.
  • All the content that we want to display to the user must be inside the < body > element.

The < h1 >, < h2 >, < h3 >, < h4 >, < h5 >, and < h6 > elements represent the level of heading a given text block represents The < p > element represent a paragraph or block of text.
The < ul >, < ol > and < li > elements represent unorder, order and list items. The < img > element contains informations about images.
The < br > element creates a line of empty space or a break line. The < a > element creates links to the same webpage or others.
The < footer > element appears at the bottom of the page.

Git

  • Git is a version control system that let us manage and keep track of each version of a project.
  • When there is a bug we use git revert to revert and fix a problem with this command.
  • Git commands always start with git to initiate or create a repo, add code and view different versions of that code.
  • To have a project save we use the cloud like Github to save our projet and changes so everything is sync with the workflow
  • There is a main branch where is the result of the project and there are feature branches where we add changes to get s final project.
  • Pull request is used to merge a comparison branch into the base branch (main).
  • We use commands like cd to access our home directory cd name of the prject and code to open it in the VS code.
  • Git status: checks what branch we are currently on.
  • Git checkout -b branch-name: creates a new branch and switches to it.
  • Git add -A command adds modifications and all the changes in the current working branch.
  • Git commit -m commit messages describe what the commit contains.
  • Git pull origin main command to have the latest version of the project.
